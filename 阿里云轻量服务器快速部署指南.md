# 阿里云轻量服务器快速部署指南

## 一、工具概况

您的工具是一个基于Flask的Python Web应用，名为"拖延探索笔记本"，具有以下特点：

- ✅ 已包含完整的Dockerfile和docker-compose.yml配置
- ✅ 已实现用户认证、数据收集、报告生成等功能
- ✅ 已配置CORS、JWT认证等安全功能
- ✅ 使用SQLite数据库，数据存储在data目录下
- ✅ 使用gunicorn作为WSGI服务器，暴露端口5001

## 二、部署准备

1. **登录阿里云轻量服务器**
   - 使用SSH工具登录到您的阿里云轻量服务器
   - 确保服务器已安装Docker和Docker Compose（您之前已安装）

2. **上传代码到服务器**
   - 使用FTP工具（如FileZilla）将您的代码上传到服务器
   - 或使用git克隆代码仓库（如果您的代码已上传到GitHub）

## 三、快速部署步骤

### 1. 进入项目目录

```bash
# 假设您的代码已上传到 /home/admin/procrastination_tool 目录
cd /home/admin/procrastination_tool
```

### 2. 修改环境变量（可选）

如果需要修改应用的密钥，可以编辑 `docker-compose.yml` 文件：

```bash
# 使用nano编辑器修改
sudo nano docker-compose.yml
```

找到 `SECRET_KEY` 行，修改为您的密钥：

```yaml
environment:
  - FLASK_ENV=production
  - SECRET_KEY=your-secret-key-here  # 修改为您的密钥
```

### 3. 启动服务

```bash
# 使用docker-compose启动服务
sudo docker-compose up -d
```

### 4. 验证部署

```bash
# 查看服务状态
sudo docker-compose ps

# 查看服务日志
sudo docker-compose logs -f
```

### 5. 访问应用

在浏览器中输入您的服务器IP地址和端口：

```
http://服务器IP:5001
```

## 四、管理与维护

### 1. 停止服务

```bash
# 停止服务但不删除容器
sudo docker-compose stop

# 停止并删除容器、网络，但保留数据卷
sudo docker-compose down
```

### 2. 更新代码

```bash
# 上传新代码或使用git pull更新代码
# 重新构建并启动服务
sudo docker-compose up -d --build
```

### 3. 查看数据

```bash
# 查看数据目录
ls -la data/

# 查看报告文件
ls -la data/reports/
```

### 4. 备份数据

```bash
# 备份数据目录
tar -czf data_backup_$(date +%Y%m%d).tar.gz data/
```

## 五、常见问题

### 1. 端口被占用

如果端口5001已被占用，可以修改 `docker-compose.yml` 文件中的端口映射：

```yaml
ports:
  - "80:5001"  # 将80端口映射到容器的5001端口
```

### 2. 数据库连接问题

确保数据目录已正确挂载：

```yaml
volumes:
  - ./data:/app/data  # 检查该挂载配置是否正确
```

### 3. 服务启动失败

查看日志以定位问题：

```bash
sudo docker-compose logs -f
```

## 六、安全建议

1. **修改默认密钥**：在 `docker-compose.yml` 中修改 `SECRET_KEY`
2. **配置防火墙**：只开放必要的端口（如5001）
3. **定期备份数据**：定期备份data目录
4. **更新依赖**：定期更新Python依赖和Docker镜像
