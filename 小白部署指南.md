## 📋 准备工作

### 1. 安装Git

#### Windows系统

1. 下载Git安装包：<https://git-scm.com/download/win>
2. 双击安装，一路点击"Next"即可（保持默认设置）
3. 安装完成后，打开命令提示符（CMD），输入`git --version`，如果显示版本号，则安装成功

#### Mac系统

1. 打开终端（Terminal）
2. 输入`xcode-select --install`，安装Xcode Command Line Tools
3. 安装完成后，输入`git --version`，如果显示版本号，则安装成功

#### Linux系统

1. 打开终端
2. 输入`sudo apt install git`（Ubuntu/Debian）或`sudo yum install git`（CentOS/RHEL）
3. 安装完成后，输入`git --version`，如果显示版本号，则安装成功

### 2. 注册GitHub账号

1. 访问GitHub官网：<https://github.com>
2. 点击"Sign up"，填写信息注册账号
3. 完成邮箱验证

### 3. 创建GitHub仓库

1. 登录GitHub
2. 点击右上角"+"号，选择"New repository"
3. 填写仓库信息：
   - Repository name: 建议填写`procrastination-tool`
   - Description: 可以填写"拖延风险评估工具"
   - Visibility: 选择"Public"（开源）
   - **License**: 选择合适的开源协议（重要！涉及数据收集）
     - 推荐选择：**MIT License** 或 **Apache License 2.0**
     - 选择理由：
       - 📝 **MIT License**：最宽松的开源协议，允许几乎任何使用、修改和分发，适合快速迭代的项目
       - 🛡️ `D:\pro`**Apache License 2.0**：与MIT类似，但包含专利保护条款，更适合涉及数据处理的项目
       - ❌ 不推荐：GPL v3/AGPL v3（强copyleft协议，要求修改后的代码也必须开源，可能限制商业使用）
   - 勾选"Add a README file"（可选）
4. 点击"Create repository"

### 4. 数据收集相关协议说明

由于本工具涉及用户数据收集（三个维度的评估数据），请特别注意：

1. **开源协议 ≠ 隐私政策**
   - 开源协议主要规定代码的使用权限
   - 隐私政策需要单独制定，明确说明：
     - 收集哪些数据
     - 数据用途是什么
     - 数据如何存储和保护
     - 用户权利（查看、修改、删除数据）
2. **服务条款制定**
   - 在用户注册/登录时必须明确展示服务条款和隐私政策
   - 获取用户的明确同意
   - 记录用户同意状态
3. **数据安全保护**
   - 采用加密存储敏感数据
   - 定期备份数据
   - 限制数据访问权限
   - 遵守相关数据保护法规（如GDPR、CCPA等）

## 📦 本地项目准备

### 1. 初始化Git仓库

1. 打开命令行工具（CMD/Terminal）
2. 进入项目目录：`cd 你的项目路径`（例如：`cd crastination_tool`）
3. 初始化Git仓库：`git init`
4. 添加所有文件到暂存区：`git add .`
5. 提交文件：`git commit -m "Initial commit"`

### 2. 关联GitHub仓库

1. 复制GitHub仓库的HTTPS链接（例如：`https://github.com/你的用户名/procrastination-tool.git`）
2. 在命令行中输入：`git remote add origin 你的GitHub仓库链接`
3. 推送代码到GitHub：`git push -u origin main`
4. 首次推送需要输入GitHub用户名和密码（或个人访问令牌）

### 3. 连接服务器

1. 进入云服务器控制台
2. 找到你的服务器，点击"远程登录"或"Web Shell"

#### 方法2：使用本地SSH工具（推荐）

1. 下载安装PuTTY（Windows）或使用终端（Mac/Linux）
2. 打开PuTTY，输入服务器的公网IP地址，端口22，点击"Open"
3. 输入用户名和密码，即可登录
   - 阿里云官方镜像默认用户名：`root`
   - Ubuntu系统默认用户名：`ubuntu`

### 4. 部署项目（Docker方式）

#### 步骤1：安装基础软件

##### 阿里云官方镜像（Alibaba Cloud Linux 3/4）

```bash
# 更新系统
sudo dnf update -y && sudo dnf upgrade -y

# 安装Git和OpenSSL
sudo dnf install -y git openssl
```

##### Ubuntu系统

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装Git（Ubuntu默认已安装OpenSSL）
sudo apt install -y git
```

#### 步骤2：安装Docker

##### 阿里云官方镜像（Alibaba Cloud Linux 3/4）

```bash
# 安装Docker
sudo dnf install -y docker docker-compose

# 启动Docker服务
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户添加到docker组（避免每次使用docker都需要sudo）
sudo usermod -aG docker $USER
newgrp docker
```

##### Ubuntu系统

```bash
# 安装Docker
sudo apt install -y docker.io docker-compose

# 启动Docker服务
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户添加到docker组（避免每次使用docker都需要sudo）
sudo usermod -aG docker $USER
newgrp docker
```

#### 步骤3：内存优化（针对2GB内存服务器）

2GB内存服务器建议添加以下优化，确保服务稳定运行：

```bash
# 配置Docker内存限制
echo '{"default-address-pools": [{"base":"172.17.0.0/16","size":24}], "default-shm-size": "256m", "live-restore": true}' | sudo tee /etc/docker/daemon.json

# 重启Docker服务
sudo systemctl restart docker
```

#### 步骤4：克隆项目

```bash
# 克隆项目到服务器
git clone https://github.com/你的用户名/procrastination-tool.git
cd procrastination-tool
```

#### 步骤5：创建环境变量文件

```bash
# 创建.env文件
cat > .env << EOL
FLASK_APP=app.py
FLASK_ENV=production
SECRET_KEY=$(openssl rand -hex 16)
DATABASE_URL=data/procrastination_notebook.db
HOST=0.0.0.0
PORT=5001
LOG_LEVEL=INFO
EOL
```

#### 步骤6：配置阿里云安全组

**阿里云服务器特有配置**：阿里云服务器默认会限制端口访问，需要在安全组中开放5001端口，否则无法访问你的网站！

##### 对于阿里云轻量应用服务器

1. 登录阿里云控制台：<https://swas.console.aliyun.com/>
2. 在左侧菜单点击"轻量应用服务器"
3. 找到你的服务器实例，点击实例名称进入详情页
4. 在左侧菜单点击"防火墙"
5. 点击"添加规则"
6. 填写规则信息：
   - 应用类型：自定义
   - 协议：TCP
   - 端口范围：5001
   - 授权对象：0.0.0.0/0（允许所有IP访问，或根据需要设置特定IP）
   - 描述：拖延风险评估工具
7. 点击"确定"保存规则

##### 对于阿里云ECS云服务器

1. 登录阿里云控制台：<https://ecs.console.aliyun.com/>
2. 在左侧菜单点击"实例与镜像" -> "实例"
3. 找到你的服务器实例，点击实例ID进入详情页
4. 在左侧菜单点击"网络与安全" -> "安全组"
5. 点击"配置规则"
6. 点击"入方向" -> "手动添加"
7. 填写规则信息：
   - 授权策略：允许
   - 协议类型：TCP
   - 端口范围：5001/5001
   - 授权对象：0.0.0.0/0（允许所有IP访问，或根据需要设置特定IP）
   - 描述：拖延风险评估工具
8. 点击"确定"保存规则

**小提示**：
- 如果不确定是轻量应用服务器还是ECS，请查看你的服务器购买页面
- 配置完成后，可能需要等待1-2分钟规则才会生效
- 可以在浏览器中输入`http://你的服务器IP:5001`测试是否能访问

#### 步骤7：启动服务

```bash
# 使用Docker Compose启动服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志（可选）
docker-compose logs -f
```

#### 步骤8：访问服务

1. 找到服务器的公网IP地址（在云服务商控制台查看）
2. 在浏览器中输入：`http://你的服务器IP:5001`
3. 看到拖延风险评估工具的登录页面，说明部署成功！

## 📋 数据收集功能说明

### 功能介绍

本工具包含数据收集功能，用于收集用户的三个维度评估数据：

1. **任务厌恶度** - 用户对任务的厌恶程度
2. **结果价值感** - 用户对任务结果的价值认知
3. **自我控制能力** - 用户的自我控制能力评估

### 同意机制

- 用户在注册/登录时必须明确同意服务条款和隐私政策
- 同意状态会被记录在数据库中
- 所有数据收集严格遵守隐私政策

```bash
sudo systemctl restart nginx
```

当你在GitHub上更新了代码，可以通过以下命令更新服务器上的项目：

```bash

git pull origin main
docker-compose down
docker-compose up -d --build
```

## 🚑 常见问题

### 1. 无法访问网站

- 检查服务器防火墙是否开放了5001端口
- 检查阿里云安全组是否配置了5001端口规则
- 检查Docker容器是否正常运行：`docker-compose ps`
- 检查日志：`docker-compose logs -f`

### 2. 忘记服务器密码

- 登录云服务商控制台，找到你的服务器
- 点击"重置密码"，按照提示操作

### 3. Git推送失败

- 检查GitHub仓库链接是否正确
- 检查GitHub用户名和密码是否正确
- 如果使用的是2FA（两步验证），需要使用个人访问令牌代替密码
