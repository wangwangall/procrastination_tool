## 一、今日完成的工作

### 1. 修复历史记录时间不一致问题

**问题描述：**
- 历史记录页面显示的时间与报告页面时间不一致
- 历史记录显示的是 UTC 时间，报告显示的是本地时间
- 两者相差 8 小时（中国时区 UTC+8）

**问题原因：**
- 数据库保存时使用 SQLite 的 `CURRENT_TIMESTAMP`（UTC时间）
- 报告生成时使用 `datetime.now()`（本地时间）

**解决方案：**
1. 修改 `app.py` 中的评估函数，显式使用本地时间保存数据库记录
2. 创建 `fix_timestamps.py` 脚本，将现有数据库记录从 UTC 转换为本地时间
3. 更新了 11 条历史记录的时间戳

**修改文件：**
- `d:/procrastination_tool/app.py` (第406-439行)
- 新增 `d:/procrastination_tool/fix_timestamps.py`

## 二、当前系统架构

### 后端 (app.py)
- **框架：** Flask + Flask-CORS
- **数据库：** SQLite (data/procrastination_notebook.db)
- **认证：** JWT + bcrypt密码加密
- **机器学习：** sklearn LogisticRegression

### 数据表结构：
1. `users` - 用户表
2. `core_function_results` - 核心函数结果
3. `model_predictions` - 模型预测记录
4. `risk_records` - 风险记录（含报告路径）
5. `feedbacks` - 用户反馈

### API端点：
- `/api/register` - 注册
- `/api/login` - 登录
- `/api/logout` - 登出
- `/api/user` - 用户信息
- `/api/assess` - 评估
- `/api/history` - 历史记录

- `/api/weights` - 权重设置
- `/api/feedback` - 提交反馈

- **风格：** 微信风格移动端UI

## 三、核心函数

```python
def get_risk_level(task_aversion, result_value, self_control, w1=0.4, w2=0.3, w3=0.3):
    score = (
        task_aversion * w1 +
        (100 - result_value) * w2 +
        (100 - self_control) * w3
    )
    if score < 30:
        risk_level = "低"
    elif score < 70:
        risk_level = "中"
    else:
        risk_level = "高"
    return risk_level, score
```
