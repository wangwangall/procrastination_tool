# 数据收集指南

## 1. 数据收集概述

本指南详细描述了拖延风险评估工具的数据收集政策，包括收集目的、范围、处理方式和用户权利。我们承诺严格保护用户隐私，所有数据收集和处理都遵循合法、正当、必要的原则。

## 2. 数据收集目的

### 2.1 改进产品功能
- 优化拖延风险评估算法
- 提升建议的针对性和有效性
- 改进用户体验
- 开发新功能和服务

### 2.2 学术研究
- 研究拖延行为的影响因素
- 探索干预方法的有效性
- 推动拖延心理学领域的发展

### 2.3 个性化服务
- 为用户提供个性化的评估结果
- 基于历史数据生成更精准的建议
- 跟踪用户进步和变化

## 4. 数据脱敏处理

### 4.1 脱敏规则
| 字段名 | 脱敏方法 | 示例 |
|-------|---------|------|
| username | 保留前2个字符，其余用*替换 | testuser → te** |
| occupation | 使用通用职业类别 | 高级软件工程师 → 程序员（脱敏） |
| age_group | 添加脱敏标记 | 25-35 → 25-35（脱敏） |

### 4.2 脱敏实现
```python
def desensitize_data(data):
    # 用户名脱敏
    if 'username' in data:
        username = data['username']
        if len(username) > 2:
            data['username'] = username[:2] + '*' * (len(username) - 2)
    
    # 职业脱敏
    if 'occupation' in data:
        sensitive_occupations = ['医生', '律师', '教师', '工程师', '程序员']
        for occ in sensitive_occupations:
            if occ in data['occupation']:
                data['occupation'] = occ + '（脱敏）'
                break
    
    # 年龄组脱敏
    if 'age_group' in data:
        data['age_group'] = data['age_group'] + '（脱敏）'
    
    return data
```

## 5. 数据存储和安全

### 5.1 存储方式
- **存储位置**：本地SQLite数据库
- **存储时间**：最长保留12个月，过期自动清理
- **备份策略**：每周自动备份，备份文件保留30天

### 5.2 安全措施
- **数据加密**：用户密码采用SHA-256加密存储
- **访问控制**：数据库文件采用权限控制，仅允许应用程序访问
- **传输安全**：所有API请求采用HTTPS加密传输
- **定期审计**：每月进行安全审计，检查数据访问日志

## 6. 用户权利

### 6.1 知情权
- 用户有权了解数据收集的目的、范围和使用方式
- 用户有权获取自己的所有数据副本

### 6.2 选择权
- 用户可以选择是否同意数据收集
- 用户可以随时撤回数据收集同意
- 不同意数据收集不影响基本功能使用

### 6.3 访问权
- 用户有权访问自己的所有评估历史
- 用户有权查看自己的数据分析结果
- 用户有权获取自己的反馈记录

### 6.4 删除权
- 用户可以随时请求删除自己的所有数据
- 删除请求将在7个工作日内处理完成
- 删除后的数据将无法恢复

### 6.5 更正权
- 用户有权更正自己的错误数据
- 更正请求将在3个工作日内处理完成

## 7. 数据使用限制

### 7.1 内部使用
- 仅用于产品改进和学术研究
- 不用于商业营销
- 不与第三方共享原始数据

### 7.2 外部共享
- 仅在用户明确同意的情况下共享数据
- 共享的数据必须经过脱敏处理
- 共享数据仅用于学术研究

### 7.3 法律要求
- 仅在法律强制要求的情况下提供数据
- 提供数据时将依法保护用户隐私

## 8. 数据收集同意

### 8.1 首次使用同意
- 用户首次使用工具时，将看到数据收集同意弹窗
- 弹窗明确说明数据收集的目的和范围
- 用户可以选择同意或不同意

### 8.2 同意撤回
- 用户可以随时在设置中撤回同意
- 撤回同意后，将不再收集新数据
- 已收集的数据将继续保留，但不再用于改进产品

## 9. 数据生命周期管理

### 9.1 数据创建
- 评估数据：用户完成评估时创建
- 行为数据：用户使用功能时创建
- 反馈数据：用户提交反馈时创建

### 9.2 数据使用
- 实时评估：用于生成当前评估结果
- 模型训练：定期用于重新训练机器学习模型
- 数据分析：用于生成用户报告和统计数据

### 9.3 数据归档
- 超过6个月的数据自动归档
- 归档数据仅用于历史分析
- 归档数据采用压缩存储

### 9.4 数据销毁
- 超过12个月的数据自动销毁
- 用户请求删除的数据立即销毁
- 销毁方式：物理删除数据库记录

## 10. 隐私保护措施

### 10.1 技术措施
- 采用HTTPS加密传输
- 数据库访问权限控制
- 定期安全漏洞扫描
- 数据脱敏处理

### 10.2 管理措施

- 定期对员工进行隐私保护培训

### 10.3 合规措施
- 遵守相关法律法规
- 接受第三方隐私认证
- 定期进行合规审计

### 13.2 术语定义
- **个人信息**：以电子或者其他方式记录的与已识别或者可识别的自然人有关的各种信息
- **敏感个人信息**：一旦泄露或者非法使用，容易导致自然人的人格尊严受到侵害或者人身、财产安全受到危害的个人信息
- **数据脱敏**：通过对敏感数据进行变形处理，使其无法识别特定个人的过程
- **同意**：自然人在充分知情的前提下，自愿、明确作出的同意数据处理的意思表示
