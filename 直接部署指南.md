# 🚀 拖延风险评估工具 - 直接部署指南（适合技术小白）

## 🌟 适用场景
- 不想使用复杂的容器化部署
- 希望快速验证服务是否能正常运行
- 技术小白，对Docker/Podman不熟悉

## 🛠️ 部署步骤

### 步骤1：查看项目目录结构

首先，确认项目中的文件：

```bash
# 查看当前目录下的所有文件
ls -la
```

**预期结果**：应该能看到 `app.py` 文件（主程序）

### 步骤2：检查是否有requirements.txt文件

如果没有requirements.txt文件，直接安装所需依赖：

```bash
# 安装Flask等基础依赖
pip3 install flask sqlite3 scikit-learn pickle-mixin
```

### 步骤3：启动服务

#### 方法1：直接启动（调试模式）

```bash
# 直接在前台运行，查看实时日志
python3 app.py
```

**预期输出**：
```
Running on http://0.0.0.0:5001/
```

#### 方法2：后台运行（生产模式）

```bash
# 在后台运行，保存日志到app.log
nohup python3 app.py > app.log 2>&1 &

# 检查服务是否启动成功
ps aux | grep app.py

# 查看日志
cat app.log
```

### 步骤4：验证服务

#### 本地验证
```bash
# 在服务器内部测试服务
curl http://localhost:5001
```

**预期结果**：返回HTML代码，说明服务正常运行

#### 外部访问
1. 登录阿里云控制台
2. 配置防火墙，开放5001端口
3. 在浏览器中访问：`http://你的服务器IP:5001`

## 📋 常见问题

### 问题1：找不到app.py文件

**解决方案**：确认是否在正确的项目目录中，重新克隆项目：

```bash
git clone https://github.com/你的用户名/procrastination-tool.git
cd procrastination-tool
```

### 问题2：缺少依赖

**解决方案**：安装所需依赖：

```bash
pip3 install flask sqlite3 scikit-learn pickle-mixin
```

### 问题3：端口被占用

**解决方案**：使用其他端口启动服务：

```bash
# 例如使用5002端口
python3 app.py --port 5002
```

### 问题4：服务启动后无法访问

**解决方案**：
1. 检查阿里云防火墙是否开放了对应端口
2. 检查服务是否运行在0.0.0.0而不是127.0.0.1
3. 查看app.log了解错误信息

## 🎯 关键注意事项

1. **确保在正确的项目目录中执行命令**
2. **直接使用Python启动是最简单可靠的方式**
3. **配置阿里云防火墙开放对应端口**
4. **使用nohup让服务在后台持续运行**
5. **查看日志了解启动过程中的错误信息**

## 🎉 部署完成

按照以上步骤操作，你应该能成功部署和访问你的服务。如果还有问题，可以查看app.log文件了解详细错误信息，或者联系开发者寻求帮助。
