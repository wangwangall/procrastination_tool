# 阿里云轻量服务器小白部署指南

## 特别说明

这份指南是专为不懂技术的小白用户编写的，每一步都有详细说明和截图指引。请严格按照步骤操作，遇到问题可以随时联系技术支持。

## 一、准备工作

### 1. 登录阿里云控制台

1. 打开浏览器，访问阿里云官网：https://www.aliyun.com/
2. 点击右上角的「登录」按钮，使用您的阿里云账号登录
3. 登录后，点击控制台左上角的「产品与服务」，选择「轻量应用服务器」

### 2. 找到您的轻量服务器

在轻量应用服务器列表中，找到您要部署应用的服务器，点击「管理」按钮进入服务器详情页。

### 3. 设置服务器密码

如果您还没有设置服务器密码：
1. 在服务器详情页，点击左侧菜单的「远程连接」
2. 点击「设置密码」，按照提示设置一个强密码（请记住这个密码，后续登录服务器需要使用）
3. 密码设置完成后，点击「重启」按钮重启服务器

## 二、登录服务器

### 1. 使用Web Shell登录

1. 在服务器详情页，点击左侧菜单的「远程连接」
2. 点击「Web Shell」按钮，在弹出的窗口中输入您刚才设置的密码
3. 点击「确定」按钮，即可登录到服务器的命令行界面

### 2. 使用SSH工具登录（推荐，更稳定）

**如果您是Windows用户：**

1. 下载并安装PuTTY工具：https://www.putty.org/
2. 打开PuTTY，在「Host Name (or IP address)」框中输入您服务器的公网IP地址
3. 在「Port」框中输入22
4. 点击「Open」按钮，在弹出的窗口中输入用户名（默认用户名是`root`）和密码
5. 点击「确定」按钮，即可登录到服务器的命令行界面

**如果您是Mac/Linux用户：**

1. 打开终端（Terminal）
2. 输入命令：`ssh root@服务器公网IP`（将「服务器公网IP」替换为您服务器的实际IP）
3. 输入密码，即可登录到服务器的命令行界面

## 三、安装必要软件

### 1. 安装Docker

在命令行中执行以下命令：

```bash
# 更新系统软件包
yum update -y

# 安装Docker
yum install -y docker

# 启动Docker服务
systemctl start docker

# 设置Docker开机自启
systemctl enable docker
```

### 2. 安装Docker Compose

在命令行中执行以下命令：

```bash
# 下载Docker Compose二进制文件
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 赋予执行权限
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

## 四、从GitHub克隆代码

### 1. 安装Git

在命令行中执行以下命令：

```bash
yum install -y git
```

### 2. 克隆您的GitHub代码

在命令行中执行以下命令（请将`your-github-username/your-repo-name`替换为您的GitHub仓库地址）：

```bash
# 进入home目录
cd /home

# 克隆代码
git clone https://github.com/your-github-username/your-repo-name.git

# 进入项目目录
cd your-repo-name
```

## 五、启动应用

### 1. 修改配置文件（可选）

如果您需要修改应用的密钥，可以执行以下命令：

```bash
# 使用nano编辑器修改docker-compose.yml文件
nano docker-compose.yml
```

找到`SECRET_KEY`行，修改为您的密钥（例如：`SECRET_KEY=my-secret-key-123`），然后按下`Ctrl+O`保存，`Ctrl+X`退出。

### 2. 启动服务

在命令行中执行以下命令：

```bash
# 使用docker-compose启动服务
docker-compose up -d
```

### 3. 验证服务状态

在命令行中执行以下命令：

```bash
# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs -f
```

如果服务状态显示`Up`，说明服务已成功启动。

## 六、访问您的应用

### 1. 查看服务器公网IP

在阿里云轻量服务器详情页，您可以看到服务器的公网IP地址。

### 2. 访问应用

在浏览器中输入您的服务器公网IP地址和端口（默认端口是5001）：

```
http://服务器公网IP:5001
```

例如：如果您的服务器公网IP是`123.45.67.89`，那么您可以在浏览器中输入`http://123.45.67.89:5001`访问您的应用。

## 七、常见问题

### 1. 无法登录服务器

- 检查服务器密码是否正确
- 检查服务器是否处于运行状态
- 检查网络连接是否正常

### 2. 克隆代码失败

- 检查GitHub仓库地址是否正确
- 检查服务器是否可以访问GitHub（可以执行`ping github.com`命令测试）
- 如果仓库是私有的，需要设置SSH密钥（请联系技术支持协助处理）

### 3. 服务启动失败

- 查看服务日志，查找错误信息：`docker-compose logs -f`
- 检查端口是否被占用：`netstat -tlnp | grep 5001`
- 检查配置文件是否正确：`cat docker-compose.yml`

### 4. 无法访问应用

- 检查服务是否正常运行：`docker-compose ps`
- 检查服务器防火墙是否开放了5001端口
  - 在阿里云轻量服务器详情页，点击左侧菜单的「防火墙」
  - 检查是否有一条规则允许TCP协议的5001端口访问
  - 如果没有，请点击「添加规则」，添加一条允许TCP协议5001端口访问的规则

## 八、数据收集

您的应用已经包含了完整的数据收集功能，所有数据都存储在服务器的`/home/your-repo-name/data`目录下，包括：

- 用户注册和登录数据
- 用户测试数据
- 自动生成的报告文件
- 风险记录数据

您可以通过以下方式查看这些数据：

```bash
# 进入项目目录
cd /home/your-repo-name

# 查看数据目录
ls -la data/

# 查看报告文件
ls -la data/reports/
```

## 九、后续维护

### 1. 更新代码

如果您在GitHub上更新了代码，可以通过以下命令更新服务器上的代码：

```bash
# 进入项目目录
cd /home/your-repo-name

# 拉取最新代码
git pull

# 重新构建并启动服务
docker-compose up -d --build
```

### 2. 备份数据

为了防止数据丢失，建议定期备份数据目录：

```bash
# 进入项目目录
cd /home/your-repo-name

# 备份数据目录
tar -czf data_backup_$(date +%Y%m%d).tar.gz data/
```

### 3. 停止服务

如果您需要停止服务，可以执行以下命令：

```bash
# 进入项目目录
cd /home/your-repo-name

# 停止服务
docker-compose stop
```
