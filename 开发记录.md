# 拖延风险评估工具开发记录

## 项目概述
基于机器学习的拖延风险评估工具，通过三个维度评估用户拖延风险并提供个性化建议。

## 开发过程

### 问题分析与解决方案
1. **文件路径错误**：用户在运行 `python app.py` 时出现文件未找到错误
   - 解决方案：将 `app.py` 从 `backend` 目录移动到根目录

2. **功能扩展**：需要在数据流和数据库中增加「任务厌恶度」维度
   - 解决方案：
     - 前端修改：在用户点击「评估风险」时，将「任务厌恶度」的值一起放进请求 body
     - 后端修改：修改 `/report` 接口，接收并存储 `task_aversion` 字段
     - 数据库修改：在 `risk_records` 表中新增 `task_aversion` 列

3. **种子数据生成**：基于时间决策理论生成训练数据
   - 解决方案：创建 `seed_data.py` 脚本，按六个场景生成50条种子数据

4. **机器学习模型**：训练模型预测拖延概率
   - 解决方案：创建 `train_model.py` 脚本，使用逻辑回归训练二分类模型

5. **前端展示**：修改前端显示预测结果
   - 解决方案：更新 `frontend/index.html`，显示拖延概率和个性化建议

6. **历史记录建议错误**：历史评估记录点进去报告后显示的建议不正确
   - 解决方案：
     - 修复 `app.py` 中的 `get_history` 函数，为每个历史记录生成正确的个性化建议
     - 确保每个记录的建议基于其自身的评估结果（任务厌恶度、结果价值、自我控制）
     - 实现动态建议生成逻辑，根据不同的风险等级返回不同的建议
     - 创建 `test_suggestion.py` 脚本验证修复效果

7. **导览整合与状态稳定**：需要修复评估导览空白问题和报告独立窗口问题
   - 解决方案：
     - 在评估导览顶部添加选项卡切换功能（"输入评估"和"完整报告"）
     - 报告直接显示在选项卡中，不再弹出新窗口
     - 使用 `localStorage` 保存评估输入状态，自动恢复
     - 添加淡入淡出动画效果，导览间切换流畅

8. **三维度评估文案优化**：需要为每个维度添加解释说明
   - 解决方案：
     - 为每个维度添加红色 ❓ 问号图标
     - 实现悬浮和点击展开的维度解释
     - 添加详细的维度定义和参考说明

9. **时间显示问题**：报告页面时间显示为 `NaN-NaN-NaN NaN:NaN`
   - 解决方案：
     - 后端修改：返回 ISO 8601 格式的时间戳
     - 前端修改：增强 `formatTime` 函数，支持多种时间格式解析

### 核心功能实现
1. **Flask后端**：
   - `/predict` 接口：预测拖延概率并返回建议
   - `/report` 接口：收集评估数据并存储到数据库
   - `/history` 接口：获取历史评估记录，包含正确的个性化建议
   - 模型加载与错误处理：实现模型加载失败时的fallback机制

2. **机器学习模型**：
   - 算法：逻辑回归（二分类）
   - 输入特征：task_aversion, result_value, self_control
   - 输出：拖延概率（0-1）
   - 准确率：约90%

3. **数据库**：
   - 存储：SQLite 数据库
   - 表结构：risk_records（id, task_aversion, result_value, self_control, actual_delay, timestamp, source）

4. **前端**：
   - 三个维度滑动条：任务厌恶度、结果价值、自我控制
   - 评估按钮：提交数据并显示结果
   - 结果展示：拖延概率和个性化建议
   - 历史记录：查看历史评估记录，点击可查看详细报告

## 运行步骤
1. **创建虚拟环境**：`conda create -n procrastinate_env python=3.8`
2. **激活环境**：`conda activate procrastinate_env`
3. **安装依赖**：`pip install flask sqlite3 scikit-learn pickle-mixin`
4. **生成种子数据**：`python seed_data.py`
5. **训练模型**：`python train_model.py`
6. **启动应用**：`python app.py`
7. **访问工具**：http://127.0.0.1:5000
