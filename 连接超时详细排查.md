# 🚀 拖延风险评估工具 - 连接超时详细排查指南

## 🌟 问题分析

用户遇到无法访问服务的问题，错误信息为 `ERR_CONNECTION_TIMED_OUT`，且已确认不是流量包问题。

## 🛠️ 排查步骤（按优先级）

### 步骤1：检查服务是否正常运行

**操作**：在服务器终端执行以下命令

```bash
# 检查服务状态（Podman）
podman-compose ps

# 或（Docker）
docker-compose ps
```

**预期结果**：服务状态显示为 `Up`

**如果服务未运行**：
```bash
# 重启服务（Podman）
podman-compose down
podman-compose up -d

# 或（Docker）
docker-compose down
docker-compose up -d
```

**查看服务日志**：
```bash
# Podman
podman-compose logs -f

# Docker
docker-compose logs -f
```

### 步骤2：检查阿里云轻量服务器防火墙（安全组）

**操作**：登录阿里云控制台，配置防火墙规则

1. 访问阿里云轻量应用服务器控制台：<https://swas.console.aliyun.com/>
2. 选择你的服务器实例
3. 点击左侧菜单的「防火墙」
4. 点击「添加规则」按钮
5. 填写以下信息：
   - 应用类型：**自定义**
   - 协议：**TCP**
   - 端口范围：**5001**
   - 授权对象：**0.0.0.0/0**（允许所有IP访问）
   - 描述：**拖延风险评估工具**
6. 点击「确定」

**等待1-2分钟**，让规则生效

### 步骤3：检查服务器内部防火墙

**操作**：在服务器终端执行以下命令

```bash
# 检查防火墙状态
sudo firewall-cmd --state

# 检查已开放的端口
sudo firewall-cmd --list-ports
```

**如果5001端口未开放**：
```bash
# 开放5001端口
sudo firewall-cmd --add-port=5001/tcp --permanent

# 重启防火墙使配置生效
sudo firewall-cmd --reload

# 再次检查已开放端口
sudo firewall-cmd --list-ports
```

### 步骤4：测试本地访问

**操作**：在服务器终端执行以下命令

```bash
# 使用curl测试本地访问服务
curl http://localhost:5001
```

**预期结果**：
- 返回HTML代码（服务正常运行）
- 如果返回错误，说明服务本身有问题

**如果返回错误**：
```bash
# 查看更详细的服务日志
podman-compose logs -f  # 或 docker-compose logs -f
```

### 步骤5：检查端口监听情况

**操作**：在服务器终端执行以下命令

```bash
# 检查5001端口是否被监听
ss -tuln | grep 5001

# 或使用netstat
netstat -tuln | grep 5001

# 或使用lsof
lsof -i :5001
```

**预期结果**：看到5001端口处于LISTEN状态

### 步骤6：检查IP地址是否正确

**操作**：确认你访问的IP地址是服务器的公网IP

```bash
# 在服务器终端查看公网IP
curl ifconfig.me
```

**对比**：确保你在浏览器中访问的IP地址与服务器公网IP一致

## 📋 常见问题及解决方案

### 问题1：服务启动失败

**症状**：`podman-compose ps` 显示服务状态为 `Exited`

**解决方案**：
- 查看日志：`podman-compose logs -f`
- 检查.env文件配置是否正确
- 检查docker-compose.yml文件配置是否正确
- 确保端口未被其他程序占用

### 问题2：安全组配置错误

**症状**：本地测试正常，但外部访问超时

**解决方案**：
- 重新检查防火墙规则
- 确保端口号正确（5001）
- 确保授权对象为0.0.0.0/0
- 等待1-2分钟规则生效

### 问题3：服务器防火墙未开放端口

**症状**：本地测试正常，但外部访问超时

**解决方案**：
- 按照步骤3开放5001端口
- 重启防火墙使配置生效

### 问题4：服务配置错误

**症状**：服务启动失败，日志中有错误信息

**解决方案**：
- 检查.env文件中的SECRET_KEY是否生成
- 检查数据库路径配置
- 检查端口配置是否为5001

## 🎯 测试方法

配置完成后，使用以下方法测试：

1. **浏览器访问**：`http://你的服务器IP:5001`
2. **命令行测试**：在本地终端执行 `curl -v http://你的服务器IP:5001`
3. **在线端口扫描**：使用<https://tool.chinaz.com/port>扫描5001端口

## 📌 注意事项

1. **安全组配置需要时间生效**：通常需要1-2分钟
2. **确保服务监听0.0.0.0**：而不是127.0.0.1
3. **检查服务是否有错误日志**：日志是排查问题的关键
4. **确保端口未被占用**：使用 `ss -tuln | grep 5001` 检查

2. 重新部署服务
