# 🤖 拖延风险评估工具 - 模型部署指南

## 📁 模型文件说明

### 1. 模型文件信息

- **文件位置**：项目根目录 `model.pkl`
- **文件大小**：739字节
- **模型类型**：机器学习模型（逻辑回归）
- **功能**：预测用户拖延概率
- **加载方式**：自动加载，无需手动干预

### 2. 模型加载机制

在 `app.py` 中，模型通过以下方式加载：

```python
# 模型文件路径
model_path = 'model.pkl'
# 加载的模型
model = None

# 加载模型函数
def load_model():
    global model
    try:
        if os.path.exists(model_path):
            with open(model_path, 'rb') as f:
                model = pickle.load(f)
            print("模型加载成功")
        else:
            print("模型文件不存在，将使用时间决策理论")
    except Exception as e:
        print(f"模型加载失败: {e}，将使用时间决策理论")
        model = None

# 应用启动时自动加载模型
load_model()
```

### 3. 模型容错机制

- ✅ **自动备份方案**：模型加载失败时，自动切换到「时间决策理论」算法
- ✅ **无中断运行**：即使模型文件缺失，工具仍能正常使用
- ✅ **详细日志**：记录模型加载状态，便于调试

## 🚀 云服务器部署时的模型处理

### 1. 模型文件上传

**自动包含**：
- 当您上传项目文件或使用Git克隆项目时，`model.pkl` 会自动包含在内
- 无需单独上传模型文件
- 模型文件路径会自动识别

**手动上传（如果需要）**：
```bash
# 使用SCP上传
scp model.pkl root@您的服务器IP:/var/www/procrastination-tool/

# 或使用SFTP工具（如WinSCP）上传到项目根目录
```

### 2. 部署脚本中的模型处理

**已包含在部署脚本中**：
- 部署脚本会自动处理模型文件
- 无需额外配置
- 模型会在应用启动时自动加载

**验证模型加载**：
```bash
# 查看应用日志，确认模型加载状态
sudo journalctl -u procrastination-tool | grep -i "模型"
```

### 3. 模型更新

**方法一：直接替换模型文件**
```bash
# 上传新模型文件，替换旧文件
scp new_model.pkl root@您的服务器IP:/var/www/procrastination-tool/model.pkl

# 重启应用，加载新模型
sudo systemctl restart procrastination-tool
```

**方法二：使用Git更新**
```bash
# 进入项目目录
cd /var/www/procrastination-tool

# 拉取最新代码（包含新模型）
git pull

# 重启应用
sudo systemctl restart procrastination-tool
```

### 4. 模型加载故障排除

**常见问题及解决方案**：

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 模型加载失败 | 模型文件损坏 | 重新上传模型文件 |
| 模型文件不存在 | 路径错误或文件缺失 | 检查模型文件路径，重新上传 |
| 权限问题 | 模型文件权限不足 | `chmod 644 /var/www/procrastination-tool/model.pkl` |
| 版本不兼容 | Python版本或库版本不匹配 | 确保服务器Python版本与训练环境一致 |

**查看详细日志**：
```bash
# 查看应用完整日志
sudo journalctl -u procrastination-tool -f

# 检查模型文件状态
ls -la /var/www/procrastination-tool/model.pkl
```

## 📱 微信小程序中的模型处理

### 1. 模型移植到小程序

**方案一：服务器端调用**
- 模型仍部署在服务器上
- 小程序通过API调用服务器获取评估结果
- 优势：模型更新方便，无需重新发布小程序
- 劣势：依赖网络连接

**方案二：小程序端部署**
- 将模型转换为小程序支持的格式（如ONNX）
- 在小程序端直接运行模型
- 优势：无需网络连接，响应更快
- 劣势：模型更新需要重新发布小程序

### 2. 微信小程序模型部署步骤

**服务器端调用方案**：
1. 部署云服务器，确保API可访问
2. 小程序通过 `wx.request()` 调用API
3. 处理API返回的评估结果
4. 展示给用户

**小程序端部署方案**：
1. 将模型转换为ONNX格式
2. 使用微信小程序的AI能力加载模型
3. 在客户端运行模型进行预测
4. 展示结果
