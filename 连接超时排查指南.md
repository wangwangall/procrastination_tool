# 🚀 拖延风险评估工具 - 连接超时排查指南

## 🌟 问题分析

从你的浏览器截图可以看到，访问 `http://8.148.7.83:5001` 时出现了 **ERR_CONNECTION_TIMED_OUT** 错误，这表示连接超时，服务器没有响应请求。

## 🛠️ 排查步骤

### 步骤1：检查服务状态

首先，确认服务是否正在运行：

```bash
# 如果使用Podman
podman-compose ps
podman-compose logs -f

# 如果使用Docker
docker-compose ps
docker-compose logs -f
```

**预期结果**：
- 服务状态显示为 `Up`
- 日志中没有明显的错误信息

### 步骤2：检查阿里云安全组（最常见原因）

阿里云服务器默认会限制端口访问，必须在安全组中开放5001端口：

#### 对于阿里云轻量应用服务器
1. 登录阿里云控制台：<https://swas.console.aliyun.com/>
2. 选择"轻量应用服务器"
3. 找到你的服务器实例，点击实例名称进入详情页
4. 左侧菜单点击"防火墙"
5. 点击"添加规则"
6. 填写规则：
   - 应用类型：自定义
   - 协议：TCP
   - 端口范围：5001
   - 授权对象：0.0.0.0/0
   - 描述：拖延风险评估工具
7. 点击"确定"

#### 对于阿里云ECS云服务器
1. 登录阿里云控制台：<https://ecs.console.aliyun.com/>
2. 选择"实例与镜像" -> "实例"
3. 找到你的服务器，点击实例ID进入详情页
4. 左侧菜单点击"网络与安全" -> "安全组"
5. 点击"配置规则"
6. 点击"入方向" -> "手动添加"
7. 填写规则：
   - 授权策略：允许
   - 协议类型：TCP
   - 端口范围：5001/5001
   - 授权对象：0.0.0.0/0
   - 描述：拖延风险评估工具
8. 点击"确定"

### 步骤3：检查服务器防火墙

确认服务器防火墙是否开放了5001端口：

```bash
# 检查防火墙状态
sudo firewall-cmd --state

# 检查已开放端口
sudo firewall-cmd --list-ports

# 如果5001端口未开放，添加端口
sudo firewall-cmd --add-port=5001/tcp --permanent
sudo firewall-cmd --reload
```

### 步骤4：检查服务器网络连通性

```bash
# 检查服务器是否能访问外部网络
ping www.baidu.com

# 检查本地端口是否在监听
ss -tuln | grep 5001
# 或
netstat -tuln | grep 5001
# 或
lsof -i :5001
```

**预期结果**：
- 能ping通百度，说明服务器网络正常
- 能看到5001端口处于LISTEN状态

### 步骤5：检查流量包情况

如果以上步骤都没问题，可能是阿里云服务器的流量包耗尽了：

1. 登录阿里云控制台
2. 进入服务器实例详情页
3. 查看"流量监控"或"网络监控"
4. 检查是否有流量包耗尽的提示
5. 如果流量包耗尽，需要购买额外的流量包

### 步骤6：测试本地访问

在服务器内部测试服务是否正常运行：

```bash
# 使用curl测试本地访问
curl http://localhost:5001
```

**预期结果**：
- 返回HTML代码，说明服务在服务器内部正常运行

## 📋 常见解决方案

### 情况1：服务未启动
```bash
# 使用Podman重启服务
podman-compose down
podman-compose up -d

# 使用Docker重启服务
docker-compose down
docker-compose up -d
```

### 情况2：安全组未配置
- 按照步骤2配置安全组，开放5001端口
- 配置后等待1-2分钟，规则才会生效

### 情况3：防火墙未开放端口
- 按照步骤3开放5001端口

### 情况4：流量包耗尽
- 购买阿里云流量包
- 或切换到"按流量计费"模式

### 情况5：服务配置错误
- 检查.env文件中的配置
- 检查docker-compose.yml文件中的端口映射

## 🎯 测试方法

配置完成后，可以使用以下方法测试：

1. **浏览器访问**：`http://你的服务器IP:5001`
2. **命令行测试**：`curl -v http://你的服务器IP:5001`
3. **在线端口扫描**：使用<https://tool.chinaz.com/port>扫描5001端口

## 📌 注意事项

1. **安全组配置需要时间生效**：通常需要1-2分钟
2. **阿里云轻量应用服务器和ECS的安全组配置位置不同**：不要搞错
3. **流量包耗尽会导致所有外部访问失败**：包括SSH连接
4. **确保服务正在监听0.0.0.0**：而不是127.0.0.1

## 🎉 排查完成

按照以上步骤排查后，应该能解决连接超时问题。如果还有问题，可以尝试重启服务器或重新部署服务。
